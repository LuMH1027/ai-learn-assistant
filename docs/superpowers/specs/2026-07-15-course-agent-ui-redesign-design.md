# 本地课程助手 UI 改版设计规格

日期：2026-07-15  
状态：用户已确认视觉方向与交互模型

## 1. 目标

本次改版面向学生日常学习，而不是一次性答辩展示。学生将课程文件放入本地目录后，最常用的路径是：选择课程、找到资料、连续提问、核对引用原文、继续追问或记录笔记。

改版目标：

- 让提问成为主工作区，减少当前界面的装饰和视觉噪声。
- 保留课程文件、课程 Agent、资料预览、摘要、练习题和笔记等现有能力。
- 让引用与原文核对始终容易到达，但允许关闭预览以放大对话区。
- 让三栏宽度可调，并在不同窗口尺寸下保持稳定、无水平溢出。
- 使用接近 Codex 的安静工具界面语言，适合长时间使用。

非目标：

- 不更换原生 HTML、CSS、JavaScript 技术栈。
- 不引入 Vue、React 或新的前端构建工具。
- 不修改 RAG、LLM、MinerU、文件存储或 HTTP API 行为。
- 不在本次改版中处理后端安全和检索质量问题。

## 2. 设计方向

### 2.1 视觉语言

采用“Codex 式浅色学习工作台”：

- 主内容以白色和近白色为主，不使用渐变背景。
- 左侧栏采用用户确认的 G“雾白灰” `#E8ECEF`。
- 主文字使用 `#20211F`，次级文字使用 `#6D747A`。
- 激活、就绪和焦点状态使用克制的绿色 `#2F7D67`。
- 引用和原文高亮使用低饱和琥珀色，不与成功状态混用。
- 错误状态使用独立红色，并同时显示文字或图标，不仅依赖颜色。
- 面板之间主要依靠细边框和背景层级区分，避免大阴影和卡片堆叠。
- 圆角不超过 `0.5rem`，正文保持系统字体，资料正文可使用系统宋体或衬线字体。

设计密度为中高密度：课程、文件和操作入口紧凑排列，但所有主要点击目标不少于 `2.75rem`。

### 2.2 页面层级

桌面端由三个功能区域组成：

1. 左栏：课程、文件树、资料目录和添加文件入口。
2. 中栏：课程标题、知识库状态、学习工具、对话记录和输入框。
3. 右栏：当前文件、回答引用和文件信息组成的资料预览。

不再使用占据大量高度的顶部品牌横幅。品牌、资料根目录和全局状态收进左栏；当前课程和知识库状态保留在中栏顶部。

## 3. 区域设计

### 3.1 左侧课程与文件栏

- 顶部显示紧凑品牌标识和“本地课程助手”。
- “课程”区域显示课程名称和文件数量，当前课程使用浅灰选中背景和绿色左边线。
- “资料”区域使用可折叠文件树，文件夹和文件均提供明确图标与键盘操作。
- 点击文件后打开右侧资料预览；若预览已关闭，则重新打开预览。
- 底部显示当前资料根目录，并提供“切换目录”和“添加课程资料”的清晰入口。
- AI、MinerU 和本地索引状态使用紧凑状态行或状态弹层展示，不占用主工作区。
- 拖拽上传保留，同时增加可点击的文件选择入口，避免只能依赖拖拽。

### 3.2 中央对话区

- 顶部显示当前课程、文件数量、知识库更新时间和“重新构建”操作。
- 工具采用标签或分段控制：对话、摘要、练习题、笔记。
- 问答模式位于工具栏右侧，保留答疑、启发、作业提示和复习模式。
- 用户消息使用浅灰气泡；助手回答使用无外框正文，减少卡片感。
- 来源显示为紧凑的引用按钮；点击后右侧预览跳转到相应页或片段。
- Agent trace 默认收起为简短状态，按需展开查看检索过程。
- 输入框固定在中栏底部，显示附件、发送状态和错误恢复入口。

### 3.3 右侧资料预览

- 面板标题明确为“资料预览”。
- 提供“当前文件 / 引用片段 / 文件信息”三个标签。
- 点击左侧文件时显示完整文件；点击回答引用时切换到引用片段并定位对应页。
- PDF、Markdown、TXT 和图片复用同一预览区域。
- PDF 无法内嵌时显示明确说明和“在新窗口打开”操作。
- 面板顶部提供“隐藏预览”按钮；关闭后中栏立即获得右栏释放的全部宽度。
- 重新打开预览时恢复此前文件、标签和滚动位置。

## 4. 百分比三栏布局

### 4.1 基本模型

所有列宽使用容器百分比，不保存像素宽度。工作台内容宽度始终为 `100%`，高度跟随动态视口单位。

默认占比：

- 左侧课程栏：`22%`
- 左分隔条：约 `0.6%`
- 中央对话区：`45.8%`
- 右分隔条：约 `0.6%`
- 右侧资料预览：`31%`

上述比例合计为 `100%`。

### 4.2 相邻区域交换规则

拖动左分隔条：

- 左栏和中栏反向交换百分比。
- 右栏占比保持不变。
- 整体宽度保持 `100%`。

拖动右分隔条：

- 中栏和右栏反向交换百分比。
- 左栏占比保持不变。
- 整体宽度保持 `100%`。

双击任一分隔条只恢复该边界两侧的默认分配，不改变不相邻区域。

### 4.3 紧凑模式与动态下限

- 左栏低于 `14%` 时进入紧凑模式。
- 紧凑模式隐藏品牌文字、课程文字、文件文字和次要计数，只保留可识别图标和工具提示。
- 左栏最小占比不写死。浏览器根据紧凑模式下最大可见元素的实际宽度，加上安全留白，再换算为容器百分比。
- 当前视觉原型在约 `915` 宽的窗口中计算出的下限约为 `5.5%`；窗口变化时重新计算。
- 中栏保留不低于约 `34%` 的可用占比，防止输入框和回答正文失去可读性。
- 右栏打开时建议限制在约 `20%` 到 `44%` 之间。

### 4.4 预览关闭与恢复

- 关闭预览时，右栏和右分隔条占比都变为 `0%`。
- 释放的比例全部分配给中栏，左栏保持不变。
- 重新打开时恢复用户上一次使用的右栏比例。
- 左栏比例、右栏比例和预览开关状态保存在浏览器本地存储中。

### 4.5 窗口响应

- 工作台宽度始终使用可用容器的 `100%`。
- 工作台高度使用动态视口单位，避免移动浏览器地址栏造成跳动。
- 窗口变化后保留列宽比例，并重新计算紧凑模式的内容下限。
- 在容器不足以同时满足三栏最小可用宽度时，默认关闭右侧预览；用户打开时以覆盖层显示。
- 在更窄的移动布局中，左栏变为导航抽屉，中栏保持主视图，预览继续使用可关闭覆盖层。

## 5. 交互状态

### 5.1 分隔条

- 鼠标悬停和拖动时使用绿色高亮，并扩大不可见命中区域。
- 使用 `col-resize` 光标。
- 分隔条具有 `role="separator"`、垂直方向说明和可读名称。
- 获取键盘焦点后，可使用左右方向键按小步长调整百分比。
- 双击恢复相应边界的默认比例。
- 拖动过程中只改变网格比例，不触发内容重排动画。

### 5.2 加载与错误

- 构建知识库、生成摘要、生成练习题和发送问题时禁用重复提交。
- 超过约 `300ms` 的操作显示明确加载状态。
- 索引过程显示当前文件和进度，不让页面看起来冻结。
- 预览失败时在右栏显示原因、重试和新窗口打开入口。
- API 错误显示在相关操作附近，Toast 只用于成功和非阻断通知。

### 5.3 动效

- 预览开关和面板状态使用约 `150–220ms` 的透明度或网格过渡。
- 拖动分隔条不使用缓动，保证边界跟随指针。
- 支持 `prefers-reduced-motion`，关闭非必要过渡。

## 6. 前端实现边界

继续使用现有文件：

- `web/index.html`：调整语义结构，增加分隔条、预览标签和紧凑状态容器。
- `web/styles.css`：重建视觉变量、百分比网格、紧凑模式、响应式布局和交互状态。
- `web/app.js`：增加预览开关、引用定位、列宽拖动、键盘调整和本地状态持久化。

现有 API 路径和响应结构保持不变。文件预览、课程切换、聊天、笔记、摘要和练习题继续调用当前后端接口。

## 7. 验证要求

功能验证：

- 左分隔条拖动时，右栏比例保持不变。
- 右分隔条拖动时，左栏比例保持不变。
- 任意拖动后总占比始终为 `100%`。
- 左栏能进入紧凑模式并达到动态内容下限。
- 预览关闭后中栏获得全部释放空间，重新打开后恢复此前比例和内容。
- 点击文件和引用均能打开正确预览。
- 刷新页面后恢复用户的列宽比例和预览状态。

视觉与可访问性验证：

- 在小手机、平板、普通笔记本和宽屏桌面尺寸下无水平溢出。
- 主要文字对比度不低于 WCAG AA 要求。
- 所有按钮、文件、课程、标签和分隔条可通过键盘到达。
- 焦点样式清晰，分隔条方向键行为可被屏幕阅读器理解。
- 最大文字缩放时不遮挡按钮或使输入框不可用。
- 开启 reduced motion 后不出现大幅运动。

回归验证：

- 现有 Python 单元测试继续全部通过。
- 使用浏览器分别验证预览打开、预览关闭、左栏紧凑、右栏加宽和移动覆盖层状态。
- 保存桌面和移动端截图，用于课程报告和验收材料。

## 8. 完成标准

当学生可以在不同窗口尺寸下稳定完成“选课程、找资料、提问、核对来源、继续追问”，且不需要与固定面板宽度对抗时，本次 UI 改版视为完成。视觉上应保持浅色、安静、工具化，不出现营销页式大标题、渐变背景、装饰性卡片堆叠或无意义动画。
