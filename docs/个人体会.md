# 个人体会

本项目围绕“本地课程资料库与 AI 学习助手”展开，最终实现了课程目录识别、资料预览、课程知识库构建、基于资料的问答、引用来源展示、课程独立会话、文件型记忆、笔记保存、摘要和练习题生成等功能。相比普通学习资源共享系统，本项目的重点不是简单上传资料，而是让课程资料可以被检索、被追问、被总结，并且回答能够回到具体来源。

实现过程中最大的难点是如何控制问答范围。通用聊天模型容易根据常识回答，但课程学习助手更需要“只基于当前课程资料回答”。因此系统将每门课程单独建立索引，问答时只检索当前课程片段，并在答案下方显示文件名、页码或片段编号。这样既降低了胡编风险，也方便学生回到原始资料复核。

另一个难点是本地部署和功能完整度之间的平衡。为了方便 Windows 本地运行，系统没有强制依赖 MySQL、Redis、Node.js 或向量数据库，而是使用 Python 标准库、JSON 索引和文件型记忆。聊天记录、课程记忆和笔记都保存为本地文件，删除文件即可清空记忆，便于演示和维护。PDF 解析方面则采用 MinerU 优先、本地 `pypdf` 降级的策略，在效果和可运行性之间取得平衡。

当前项目仍有不足。检索已经具备 BM25、RRF、持久向量索引、本地 rerank 和 MMR 组合，但仍使用本地 JSON 文件而不是独立向量数据库，真实语义召回效果也依赖 embedding 配置；DOCX 已支持基础文本抽取，但复杂版式、扫描件和图片文字仍会受解析质量限制；PDF 如果页数过多或扫描质量较差，解析效果也会受到影响。后续可以继续增强 OCR 版面解析、multi-vector 检索和更完善的错题本功能。

通过本次课程设计，我对 RAG 系统的完整链路有了更具体的理解：资料解析、文本切片、索引构建、课程隔离、引用追踪、提示词组织和会话记忆不是孤立功能，而是共同决定 AI 学习助手是否可信、是否可用的关键环节。
