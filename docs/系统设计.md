# 系统设计

本文用于课程设计报告的“需求分析、概要设计、详细设计”章节，可直接引用其中的图表和说明。

## 1. 总体架构图

```mermaid
flowchart LR
    User["学生/使用者"] --> Browser["浏览器前端<br/>HTML/CSS/JavaScript"]
    Browser --> API["Python 本地 HTTP 服务"]

    API --> Scanner["课程扫描模块<br/>scanner.py"]
    API --> Parser["资料解析模块<br/>parser.py + MinerU"]
    API --> RAG["课程 RAG 检索模块<br/>rag.py"]
    API --> LLM["Kimi 单模型调用<br/>llm.py"]
    API --> Store["文件型会话记忆<br/>store.py"]
    API --> Upload["拖拽上传模块<br/>uploads.py"]

    Scanner --> CourseFiles["本地课程资料目录"]
    Parser --> CourseFiles
    Upload --> CourseFiles
    Upload --> ChatUploads["聊天临时附件<br/>data/chat_uploads/"]
    RAG --> Index["课程索引<br/>data/indexes/*.json"]
    Store --> MemoryFiles["课程记忆文件<br/>data/course_memory/"]
    LLM --> Kimi["硅基流动 Kimi-K2.6"]
    Parser --> MinerU["MinerU API<br/>可选"]
```

设计说明：

- 系统采用本地 Web 方式运行，用户通过浏览器访问 `http://127.0.0.1:8000`。
- 后端使用 Python 标准库 HTTP 服务，避免复杂部署。
- 课程资料来自本地文件夹，一级文件夹识别为课程。
- 每门课程拥有独立文件树、独立知识库、独立聊天记录和独立记忆。
- 大模型只使用一套 Kimi 配置，文本问答和截图问答都走同一个 OpenAI-compatible 接口。
- 运行数据保存在 `data/` 下，真实配置和聊天记录不会提交到仓库。

```mermaid
        float score
    }
    COURSE_MEMORY {
        string course_id
        text memory_md
    }
    NOTE {

        string title
        text content
        datetime created_at
    }
    STUDY_ARTIFACT {
        string type
        string markdown_path
        datetime created_at
    }
```

实现说明：

- `COURSE` 和 `FILE_NODE` 由本地目录实时扫描生成。
- `DOCUMENT_CHUNK` 存储在 `data/indexes/<course_id>.json`。
- `CHAT_MESSAGE` 存储在 `data/course_memory/<course_id>/messages.json`。
- `COURSE_MEMORY` 存储在 `data/course_memory/<course_id>/memory.md`。
- `NOTE` 存储在 `data/course_memory/<course_id>/notes.json`。
- `STUDY_ARTIFACT` 对应课程目录下的 `AI生成/*.md`。

## 5. 关键流程图

### 5.1 课程扫描流程

```mermaid
flowchart TD
    A["输入资料根目录"] --> B{"目录是否存在"}
    B -- 否 --> C["返回错误提示"]
    B -- 是 --> D["遍历一级子文件夹"]
    D --> E["每个一级文件夹作为课程"]
    E --> F["递归扫描课程内部文件"]
    F --> G{"是否为支持类型"}
    G -- 否 --> H["忽略"]
    G -- 是 --> I["生成文件节点"]
    I --> J["返回课程树"]
```

### 5.2 知识库构建流程

```mermaid
flowchart TD
    A["选择课程"] --> B["点击构建知识库"]
    B --> C["清空该课程旧索引"]
    C --> D["遍历课程资料文件"]
    D --> E{"文件类型"}
    E -- PDF --> F["优先 MinerU 解析<br/>失败降级 pypdf"]
    E -- TXT/MD --> G["直接读取文本"]
    E -- 图片 --> H["保存为图片附件说明"]
    F --> I["文本切片"]
    G --> I
    H --> I
    I --> J["写入 data/indexes/course_id.json"]
    J --> K["返回文件数和片段数"]
```

### 5.3 Agent 问答流程

```mermaid
flowchart TD
    A["用户输入问题"] --> B["可选拖入附件/截图"]
    B --> C["保存用户消息"]
    C --> D["解析聊天附件"]
    D --> E["当前课程索引检索"]
    E --> F{"是否有截图"}
    F -- 是 --> G["将截图和检索结果发给 Kimi"]
    F -- 否 --> H["将引用片段发给 Kimi"]
    G --> I["生成回答"]
    H --> I
    I --> J["按模式添加答疑/启发/作业/复习提示"]
    J --> K["保存助手消息、引用、Agent trace"]
    K --> L["更新课程 memory.md"]
    L --> M["前端显示完整聊天记录"]
```

### 5.4 摘要与练习题生成流程

```mermaid
flowchart TD
    A["点击生成摘要/练习题"] --> B["读取当前课程索引"]
    B --> C{"索引是否为空"}
    C -- 是 --> D["提示先构建知识库"]
    C -- 否 --> E["按片段生成 Markdown 内容"]
    E --> F["写入课程目录 AI生成/*.md"]
    F --> G["保存一条助手消息"]
    G --> H["刷新课程文件树"]
    H --> I["用户可点击预览生成文件"]
```

## 6. 前后端接口划分

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | `/api/config` | 读取资料根目录和配置状态，不返回密钥 |
| POST | `/api/config` | 设置资料根目录 |
| GET | `/api/courses` | 扫描课程列表和文件树 |
| GET | `/api/files/preview?id=` | 预览 PDF、文本、图片 |
| POST | `/api/courses/{course_id}/index` | 构建课程知识库 |
| GET | `/api/courses/{course_id}/messages` | 获取课程聊天记录 |
| POST | `/api/courses/{course_id}/chat` | 课程问答，支持附件和截图 |
| GET | `/api/courses/{course_id}/memory` | 获取课程记忆文件内容 |
| POST | `/api/courses/{course_id}/summary` | 生成摘要并保存到课程目录 |
| POST | `/api/courses/{course_id}/quiz` | 生成练习题并保存到课程目录 |
| GET | `/api/courses/{course_id}/notes` | 获取课程笔记 |
| POST | `/api/courses/{course_id}/notes` | 保存课程笔记 |

## 7. 模块划分

| 模块 | 文件 | 职责 |
| --- | --- | --- |
| 课程扫描 | `scanner.py` | 将一级文件夹识别为课程，生成多级文件树 |
| 文档解析 | `parser.py`、`mineru_api.py` | 解析 PDF/文本/图片，优先 MinerU，失败降级 |
| RAG 检索 | `rag.py` | 文本切片、关键词评分、引用生成、摘要和练习题生成 |
| 大模型调用 | `llm.py` | 调用 Kimi OpenAI-compatible 接口，支持文本和图片内容块 |
| 文件记忆 | `store.py` | 用 JSON/Markdown 保存聊天、记忆、笔记 |
| 上传管理 | `uploads.py` | 保存课程资料拖拽上传和聊天临时附件 |
| HTTP 服务 | `server.py` | 路由 API、协调扫描、解析、检索、生成和预览 |
| 前端交互 | `web/` | 三栏 Agent 工作台、文件树、预览、聊天、笔记抽屉 |
