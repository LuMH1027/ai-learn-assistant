# 报告撰写指南

本文用于课程设计报告撰写，按照老师给出的要求组织章节和图表。

## 1. 封面

填写课程设计题目：

```text
基于 RAG 的 Windows 本地课程资料库与 AI 学习助手系统设计与实现
```

## 2. 目的要求

写明项目目标：

- 实现一个可在 Windows 本地部署的学习资料管理系统。
- 根据本地文件夹结构自动识别课程。
- 支持 PDF 预览和课程资料浏览。
- 支持每门课程独立知识库、独立会话和独立记忆。
- 实现带引用来源的课程问答，降低 AI 幻觉。
- 支持课程笔记保存，形成个人学习空间。

## 3. 可行性分析

建议从四方面写：

- 技术可行性：Python、浏览器、SQLite、本地文件系统都成熟稳定。
- 经济可行性：系统可本地运行，不依赖昂贵服务器。
- 操作可行性：用户只需选择资料根目录，按课程展开文件即可。
- 时间可行性：先完成个人本地版，再扩展为多角色课程平台。

## 4. 需求分析

功能需求：

- 资料根目录设置。
- 课程自动识别。
- 文件树浏览。
- PDF 在线预览。
- 课程知识库构建。
- 课程 Agent 问答。
- 引用来源展示。
- 课程会话和记忆隔离。
- 课程笔记管理。
- 自动生成课程摘要和练习题。

非功能需求：

- 本地部署。
- 操作简单。
- 数据可持久化。
- 课程之间数据隔离。
- 无依据时不编造答案。

本章必须绘制：

- 数据流图。
- 用例图。

## 5. 概要设计

建议画系统结构图：

```text
浏览器前端
  ├─ 课程文件树
  ├─ PDF 预览区
  └─ 课程 Agent 会话区
        │
Python 本地服务
  ├─ 课程扫描模块
  ├─ 文档解析模块
  ├─ RAG 检索模块
  ├─ 会话记忆模块
  └─ 文件预览接口
        │
本地存储
  ├─ 本地课程资料文件夹
  ├─ SQLite 数据库
  └─ 课程知识库索引
```

说明前后端接口划分：

- `/api/config`
- `/api/courses`
- `/api/files/preview`
- `/api/courses/{course_id}/index`
- `/api/courses/{course_id}/chat`
- `/api/courses/{course_id}/memory`
- `/api/courses/{course_id}/summary`
- `/api/courses/{course_id}/quiz`
- `/api/courses/{course_id}/notes`

## 6. 详细设计

本章必须包含：

- E-R 图。
- 关键流程图。

E-R 图实体建议：

- Course：课程。
- FileNode：文件节点。
- DocumentChunk：文档片段。
- ChatMessage：聊天消息。
- Citation：引用来源。
- CourseMemory：课程记忆。
- Note：学习笔记。

关键流程图建议：

1. 课程扫描流程：输入根目录 → 遍历一级文件夹 → 过滤支持文件 → 生成课程文件树。
2. PDF 预览流程：点击文件 → 请求预览接口 → 浏览器加载 PDF。
3. 知识库构建流程：选择课程 → 解析文件 → 文本切片 → 建立索引。
4. Agent 问答流程：用户提问 → 课程内检索 → 生成回答 → 添加引用 → 更新课程记忆。

## 7. 程序模块

不要整页粘贴代码。建议只展示半页核心代码，例如：

- `CourseKnowledgeBase.answer()` 中“检索、生成引用、无依据提示”的逻辑。
- 或 `CourseScanner.scan()` 中“一级文件夹识别课程”的逻辑。

文字重点说明模块职责：

- `scanner.py`：扫描课程目录。
- `parser.py`：抽取文档文本。
- `rag.py`：课程知识库和检索问答。
- `store.py`：保存会话、记忆、笔记。
- `server.py`：提供本地 HTTP 接口。
- `llm.py`：可选 Ollama 调用和基于引用的提示词构造。

## 9. 个人体会

建议聚焦项目本身：

- 已实现：本地课程识别、PDF 预览、课程独立问答、引用来源。
- 难点：PDF 文本抽取、课程隔离、无依据回答控制。
- 不足：当前检索为轻量实现，后续可接入 MinerU、ChromaDB、Ollama。
- 收获：理解了 RAG 系统中文档解析、切片、检索、引用和会话记忆之间的关系。
