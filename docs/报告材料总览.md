# 报告材料总览

本文对应老师截图中的五项要求，列出已经准备好的材料和所在文件。

## 1. 报告结构要求

可用文件：

- `docs/报告撰写指南.md`
- `docs/系统设计.md`
- `README.md`

推荐报告题目：

```text
基于 RAG 的 Windows 本地课程资料库与 AI 学习助手系统设计与实现
```

推荐九章结构：

1. 封面。
2. 目的要求。
3. 可行性分析。
4. 需求分析。
5. 概要设计。
6. 详细设计。
7. 程序模块。
8. 测试与用户手册。
9. 个人体会。

格式提醒：

- 图表编号统一，如“图 4-1 数据流图”“表 8-1 测试用例表”。
- 参考文献放最后，正文引用格式统一。
- 代码最多展示半页，重点解释逻辑，不整页贴代码。

## 2. 关键章节要点

需求分析应包含：

- 功能需求：课程识别、文件预览、知识库构建、课程问答、引用来源、附件/截图问答、笔记、摘要和练习题生成。
- 非功能需求：Windows 本地运行、低部署成本、课程隔离、文件型记忆、可追溯回答、配置安全。
- 图：数据流图、用例图。

概要设计应包含：

- 系统结构图。
- 前后端接口划分。
- 存储结构说明。
- Kimi、MinerU 与本地降级策略。

详细设计应包含：

- E-R 图。
- 课程扫描流程图。
- 知识库构建流程图。
- Agent 问答流程图。
- 摘要/练习题生成流程图。

上述图已经写在：

```text
docs/系统设计.md
```

## 3. 程序模块与代码

可用文件：

```text
docs/程序模块与代码摘录.md
```

建议展示的核心代码：

- `scanner.py`：一级文件夹识别课程。
- `rag.py`：文本切片、关键词检索和引用生成。
- `server.py`：课程问答和附件/截图处理。
- `store.py`：文件型聊天记录、记忆和笔记。

不要贴整页代码，每段控制在 20 到 35 行以内。

```text

```

写作重点：
