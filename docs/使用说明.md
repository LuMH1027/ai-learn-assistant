# 使用说明

本文可作为课程设计报告中的“用户手册”章节。

## 1. 环境准备

必需环境：

- Windows 10/11、macOS 或 Linux。
- Python 3.9 或以上。
- Chrome、Edge 或其他现代浏览器。
- 一个本地课程资料文件夹。

建议安装依赖：

```bat
py -m pip install -r requirements.txt
```

macOS/Linux：

```bash
python3 -m pip install -r requirements.txt
```

`requirements.txt` 当前只包含 `pypdf`，用于 MinerU 不可用时本地抽取 PDF 文本。

## 2. 准备配置

仓库提供脱敏配置示例：

```text
data/config.example.json
```

复制为真实本地配置：

```bat
copy data\config.example.json data\config.json
```

macOS/Linux：

```bash
cp data/config.example.json data/config.json
```

打开 `data/config.json`，修改：

- `root_folder`：课程资料根目录。
- `ai.api_key`：硅基流动 API Key。
- `ai.model`：`Pro/moonshotai/Kimi-K2.6`。
- `mineru.token`：MinerU token。

真实配置文件位于 `data/config.json`，已经被 `.gitignore` 忽略，不应提交。

## 3. 准备课程资料

资料根目录下的一级文件夹就是课程名，例如：

```text
D:\StudyMaterials
├─ 操作系统
│  ├─ 教材
│  │  └─ chapter1.pdf
│  ├─ 课件
│  │  └─ process.pdf
│  └─ notes.md
├─ 数据结构
│  ├─ stack.pdf
│  └─ queue.txt
└─ 高等数学
   └─ limit.pdf
```

注意：

- PDF 建议小于 20 页，当前系统仍处于开发测试阶段。
- 课程资料可以包含子文件夹。
- 支持 PDF、TXT、MD、Markdown、DOCX 文件名识别，以及 PNG/JPG/WebP/GIF/BMP 图片预览。

## 4. 启动系统

Windows：

```bat
start.bat
```

macOS/Linux：

```bash
python3 run.py
```

浏览器访问：

```text
http://127.0.0.1:8000
```

截图参考：`docs/screenshots/01-首页总览.png`

## 5. 设置资料根目录

在顶部“资料根目录”输入框填写目录，例如：

```text
D:\StudyMaterials
```

点击“设置目录”或“刷新课程”。左侧会显示课程列表。

## 6. 浏览课程与文件

1. 点击左侧课程名。
2. 展开课程内部目录结构。
3. 点击 Markdown/TXT/图片文件，在预览区查看。
4. 点击 PDF 文件，浏览器会打开独立 PDF 预览窗口。

截图参考：

- `docs/screenshots/02-课程展开与文件树.png`
- `docs/screenshots/03-文本资料预览.png`

## 7. 构建课程知识库

选择课程后，点击“构建知识库”。系统会：

1. 清空当前课程旧索引。
2. 解析课程资料。
3. 将文本切片。
4. 写入 `data/indexes/<course_id>.json`。

每门课程单独构建知识库，避免不同课程资料混淆。

## 8. 使用课程 Agent

在右侧课程 Agent 中输入问题，点击“发送”。

支持模式：

- 答疑模式：直接回答。
- 启发模式：先给学习提示。
- 作业提示：只给思路和资料依据。
- 复习模式：适合整理重点和自测题。

回答会包含：

- 基于当前课程资料的答案。
- 来源文件、页码或片段编号。
- Agent trace：感知、读取、检索、回答、记忆。

截图参考：`docs/screenshots/04-课程Agent问答记录.png`

## 9. 拖入附件或截图提问

拖文件到聊天框后，系统会保存到：

```text
data/chat_uploads/<course_id>/
```

支持：

- PDF、Markdown、TXT 等文本资料。
- PNG、JPG、WebP、GIF、BMP 截图。

文本附件会被解析并加入当前课程检索；截图会作为图片内容块发送给 Kimi。

## 10. 生成摘要和练习题

点击：

- “生成摘要”
- “生成练习题”

系统会生成 Markdown 文件并保存到当前课程目录：

```text
课程目录/AI生成/
```

生成后文件会出现在左侧课程文件树中，可以点击预览，也可以重新构建知识库后纳入后续问答。

## 11. 保存课程笔记

点击右侧“笔记”按钮，打开课程笔记抽屉。

笔记保存到：

```text
data/course_memory/<course_id>/notes.json
```

截图参考：`docs/screenshots/05-课程笔记抽屉.png`

## 12. 清空课程记忆

每门课程的聊天和记忆都是文件：

```text
data/course_memory/<course_id>/
├─ messages.json
├─ memory.md
└─ notes.json
```

想清空某门课程记忆，删除对应文件或整个课程文件夹即可。
