# 使用说明

本文面向使用和验收人员，说明如何准备资料、启动系统、完成课程浏览、PDF 预览和课程 Agent 问答。

## 1. 环境要求

- 操作系统：Windows 10/11。
- Python：建议 Python 3.9 或更高版本。
- 浏览器：Chrome、Edge 或其他现代浏览器。
- 可选依赖：`pypdf`，用于抽取 PDF 文本。如果没有安装，PDF 仍可预览，但知识库问答效果会受影响。
- 可选增强：Ollama，用于本地大模型回答。
- 可选增强：MinerU Agent 轻量解析 API，用于更强的 PDF 解析。

检查 Python：

```bat
python --version
```

## 2. 准备课程资料

建立一个资料根目录，根目录下的一级文件夹就是课程名。

示例：

```text
D:\StudyMaterials
├─ 操作系统
│  ├─ 第一章.pdf
│  ├─ 第二章.pdf
│  └─ 复习笔记.md
├─ 数据结构
│  ├─ 栈和队列.pdf
│  └─ 习题.txt
└─ 计算机网络
   └─ TCP-IP.pdf
```

注意：

- 不要把 PDF 直接放在根目录下，应放进课程文件夹。
- 课程文件夹内部可以继续分子文件夹。
- 视频、压缩包等非学习文本资料会被忽略。

## 3. 启动系统

双击项目根目录下的：

```text
start.bat
```

看到类似输出后表示启动成功：

```text
Local Course Agent running at http://127.0.0.1:8000
```

在浏览器打开：

```text
http://127.0.0.1:8000
```

## 4. 设置资料根目录

在页面顶部输入资料根目录，例如：

```text
D:\StudyMaterials
```

点击“设置目录”。

如果目录正确，左侧会出现课程列表；如果没有课程，请检查根目录下是否存在一级课程文件夹。

## 5. 可选设置 AI 和 MinerU

页面第二行只显示配置状态。AI 和 MinerU 配置写在本地文件：

```text
data/config.json
```

Ollama 配置示例：

```json
{
  "ai": {
    "provider": "ollama",
    "ollama_url": "http://127.0.0.1:11434",
    "ollama_model": "qwen2.5:7b"
  }
}
```

填写后，系统会先从当前课程知识库中检索相关片段，再调用 Ollama 基于这些片段生成回答。这样既能保持引用来源，又能让答案更自然。

MinerU 默认自动开启，会优先使用 MinerU Agent 轻量解析 API。若网络不可用、文件过大或解析失败，系统会自动降级到普通 PDF 文本抽取。

## 6. 浏览课程和文件

1. 点击左侧课程名。
2. 课程会展开内部文件树。
3. 点击 PDF 文件，系统会打开独立预览窗口。
4. 点击 TXT/MD 文件，中间区域显示文本内容。
5. 可以把文件拖到左侧课程资料区，加入当前课程目录。

## 7. 构建课程知识库

选择课程后，点击左侧顶部“构建知识库”。

系统会读取该课程下支持的文件，并生成课程专属索引。构建完成后会提示入库文件数量和片段数量。

每门课程需要单独构建知识库。这样可以保证不同课程之间的问答和记忆互不影响。

## 8. 使用课程 Agent

选择课程并构建知识库后，在右侧输入问题，点击“发送”。

支持模式：

- 答疑模式：直接基于课程资料回答。
- 启发模式：先给提示，适合自学。
- 作业提示：只给思路和资料依据，不直接替用户完成作业。
- 复习模式：适合生成复习提纲、重点和题目。

也可以把 PDF、Markdown 或 TXT 拖进聊天框，作为临时附件让 AI 阅读。聊天附件不会写入课程目录，但会参与当前课程检索。

右侧还提供两个快捷按钮：

- 生成摘要：根据当前课程知识库生成复习摘要。
- 生成练习题：根据当前课程知识库生成自测题和参考要点。

回答下方会显示引用来源，例如：

```text
来源：第一章.pdf 第 3 页，片段 2
```

点击引用可以回到对应文件预览。

## 9. 保存课程笔记

右侧“课程笔记”区域可以保存：

- 本节重点。
- 错题原因。
- AI 回答中的关键结论。
- 下一次复习计划。

笔记按课程单独保存，切换课程后只显示当前课程的笔记。

## 10. 常见问题

### 页面没有课程

确认输入的是资料根目录，不是某一门课程目录。正确结构是“根目录/课程名/资料文件”。

### PDF 能预览但问答没有内容

可能是 PDF 扫描版图片，普通文本抽取不到内容。后续可接入 MinerU OCR/版面解析增强。

### 为什么回答不像大模型那么自然

当前版本是本地轻量原型，使用检索片段生成可追溯回答，未强制依赖云端模型。后续可接入 Ollama 或云 API 增强生成效果。

如果已经安装 Ollama，可以在 `data/config.json` 里填写 Ollama 地址和模型名。系统会保持“先检索课程资料，再基于引用片段回答”的策略。

### 如何重建知识库

重新点击当前课程的“构建知识库”即可覆盖旧索引。
