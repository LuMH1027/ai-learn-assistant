# 验收演示脚本

本文用于第四周线上验收演示。建议演示时间控制在 8 到 12 分钟，重点展示系统真实运行效果。

## 1. 开场说明

项目名称：基于 RAG 的 Windows 本地课程资料库与 AI 学习助手系统设计与实现。

项目定位：面向学生本地课程资料管理和复习问答，解决资料分散、查找困难、AI 回答不可追溯的问题。

团队分工：杨哲涵担任组长，负责进度统筹、接口联调、验收演示和材料提交；陆铭皓负责 RAG 核心、知识库构建和模型接入；王子欣负责前端交互、资料预览、需求测试和用户手册；徐若轩负责学习功能、设计图表、测试和报告格式。

## 2. 启动系统

演示操作：

```bash
./start.sh
```

Windows 可演示：

```bat
start.bat
```

说明要点：

- 启动脚本会构建 Vue 前端并启动 Python 本地服务。
- 默认访问地址为 `http://127.0.0.1:8000`。
- 真实配置使用 `data/config.json`，提交时只保留脱敏示例 `data/config.example.json`。

## 3. 展示课程资料管理

演示操作：

1. 打开系统首页。
2. 展示资料根目录。
3. 点击“刷新课程”。
4. 选择“操作系统”或“数据结构”课程。
5. 展开课程文件树。

说明要点：

- 系统按照一级文件夹识别课程。
- 每门课程都有独立文件树、学习计划、聊天记录和知识库。

## 4. 展示资料预览

演示操作：

1. 点击 Markdown 或 TXT 文件。
2. 展示右侧预览区。
3. 如果有 PDF 或图片资料，可点击展示预览效果。

说明要点：

- 支持 PDF、图片、TXT、Markdown 和 DOCX 入库。
- Markdown 会渲染排版，TXT/Markdown 支持文件内搜索。

## 5. 展示知识库构建

演示操作：

1. 点击“构建知识库”。
2. 展示构建完成后的文件数、片段数或状态提示。

说明要点：

- 构建过程包括解析、质量评估、切片、JSON 索引和向量索引。
- 不同课程分别构建索引，避免资料混淆。

## 6. 展示课程 Agent 问答

演示问题示例：

```text
页表有什么作用？
```

说明要点：

- 回答基于当前课程资料。
- 答案下方展示引用来源，如文件名、页码或片段编号。
- 无资料依据时系统会提示无法找到可靠依据。

## 7. 展示学习功能

演示操作：

1. 点击“生成摘要”，展示课程目录 `AI生成/` 中的 Markdown 文件。
2. 点击“生成练习题”，展示生成的自测题。
3. 打开“课程笔记”，新增或查看笔记。
4. 展示左侧学习计划和课程概览。
5. 展示掌握度和错题订正区域。

说明要点：

- 摘要和练习题作为学习产物保存到课程目录。
- `AI生成/` 在构建知识库时会被排除，避免生成内容污染证据。
- 笔记、学习计划、掌握度均按课程独立保存。

## 8. 展示测试与工程质量

可说明已准备的验证命令：

```bash
npm test --prefix frontend -- --run
npm run typecheck --prefix frontend
npm run build --prefix frontend
python3 -m unittest discover -s tests -v
```

说明要点：

- 仓库包含前端组件、API、后端服务、RAG、解析、引用、备份等测试。
- 本次整理材料时已运行前端构建，构建通过。

## 9. 结束总结

总结项目已实现内容：

- 本地课程资料识别和预览。
- 课程独立知识库和引用问答。
- 摘要、练习题、笔记、学习计划和掌握度。
- 配置诊断、RAG 评测和备份恢复。

说明不足和后续方向：

- 复杂扫描 PDF 和图片文字仍依赖 OCR 能力。
- 真实语义召回效果依赖 embedding 配置。
- 后续可继续完善错题本、复习推荐和前端备份入口。
