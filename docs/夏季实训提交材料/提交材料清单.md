# 提交材料清单

## 1. 电子代码

应包含：

- `local_course_agent/`、`frontend/`、`scripts/`、`tests/`。
- `sample_materials/` 和必要的 `test_materials/` 说明。
- `README.md`、`docs/`、安装与启动脚本。
- `requirements.txt`、`frontend/package.json`、`frontend/package-lock.json`。
- 脱敏的 `data/config.example.json`。

不得包含：

- `data/config.json`、真实 API key/token、个人资料。
- `data/course_memory/`、`data/chat_uploads/`、`data/indexes/` 等运行数据。
- `.venv/`、`frontend/node_modules/`、`web/dist/` 和缓存。

## 2. 报告材料

- 主报告：`docs/夏季实训提交材料/大型实习报告.md`
- 系统设计：`docs/夏季实训提交材料/系统设计.md`
- 代码摘录：`docs/夏季实训提交材料/程序模块与代码摘录.md`
- 用户手册：`docs/夏季实训提交材料/使用说明.md`
- 验收清单：`docs/夏季实训提交材料/验收清单.md`
- 过程材料：同目录的分工、日志和演示脚本。
- 当前验证：`docs/acceptance-report.md`

## 3. 图表和截图

设计图位于系统设计和大型报告；截图位于 `docs/screenshots/`，共 5 张。确认它们展示当前多对话 UI，不含已移除的学习计划/课程概览界面，也不泄露密钥和私人路径。

## 4. 提交前检查

- 补全封面班级、学号、指导教师和日期。
- 检查图表编号、标题层级、字体字号、页码和参考文献。
- 确认代码摘录来自当前文件且每段不超过半页。
- 确认测试表与 `acceptance-report.md` 一致，未覆盖的外部服务明确标注。
- 运行 `git diff --check` 和 `git status --short`，检查敏感文件与构建产物。
- 由组长统一提交电子代码和大型实习报告。
