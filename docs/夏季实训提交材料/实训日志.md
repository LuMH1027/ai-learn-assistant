# 实训日志

## 第一周：选题与可行性分析

本周确定项目题目为“基于 RAG 的 Windows 本地课程资料库与 AI 学习助手系统设计与实现”。小组讨论后认为，学生课程资料分散、复习时难以快速定位知识点，普通聊天模型又容易脱离课程资料回答，因此决定开发一个本地课程资料工作台。

完成内容：

- 明确项目类型为 Web 应用与 AI 应用结合。
- 完成可行性分析，从技术、经济、操作和时间四方面评估。
- 梳理功能需求和非功能需求。
- 确定四人分工，杨哲涵担任组长，陆铭皓负责 RAG 核心，王子欣负责前端与测试，徐若轩负责学习功能与设计材料。

## 第二周：基础功能实现

本周重点完成系统主流程。小组共同实现课程资料根目录扫描、一级文件夹课程识别和前端三栏布局，包括课程列表、文件树、课程对话和资料预览区域。

完成内容：

- 陆铭皓实现课程扫描、基础索引和问答接口。
- 王子欣实现文件树交互和 PDF、图片、TXT、Markdown 预览页面。
- 杨哲涵参与接口联调和配置诊断流程检查。
- 搭建 Python 本地 HTTP 服务和 Vue 前端工作台。
- 完成基础配置读取、资料根目录设置和课程切换。

## 第三周：RAG 与学习功能扩展

本周重点开发课程知识库和 AI 学习功能。系统对每门课程独立解析资料、切片、建立索引，并在问答时展示引用来源。为了提升召回效果，加入 BM25、RRF、向量索引、本地 rerank 和 MMR 去重，同时补充摘要、练习题、学习计划和错题订正等学习模块。

完成内容：

- 陆铭皓实现课程知识库构建、RAG 检索、引用来源和处理 trace。
- 王子欣完善聊天附件、截图问答入口和前端测试。
- 徐若轩实现摘要、练习题、学习计划、掌握度和错题订正相关功能与设计说明。
- 杨哲涵完善配置健康状态、备份恢复检查和演示联调。

## 第四周：测试、报告与验收准备

本周重点进行验收准备。小组按照老师要求整理大型实习报告九章结构，补充数据流图、用例图、系统结构图、E-R 图和关键流程图，并准备真实功能截图。

完成内容：

- 整理 `docs/大型实习报告.md`。
- 整理 `docs/团队分工与进度.md`、`docs/验收演示脚本.md` 和 `docs/提交材料清单.md`。

- 核对报告格式、测试用例、用户手册和个人体会。
