# 使用说明

本文可作为课程设计报告中的“用户手册”章节。

## 1. 环境准备

必需环境：

- Windows 10/11、macOS 或 Linux。
- Python 3.9 或以上。
- Node.js 20.19+ 或 22.12+，并包含 npm。
- Chrome、Edge 或其他现代浏览器。
- 一个本地课程资料文件夹。

一键安装 Python 与前端依赖：

```bat
install-deps.bat
```

macOS/Linux：

```bash
chmod +x install-deps.sh start.sh
./install-deps.sh
```

安装器会创建 `.venv`，安装 `requirements.txt` 中用于 PDF 降级解析的 `pypdf`，并通过 `frontend/package-lock.json` 安装 Vue、Vite、TypeScript、Pinia、Vitest 等 npm 依赖。安装失败时不要改用无锁的 `npm install`，应先根据错误信息检查网络、Node 版本和 npm 镜像。

## 2. 准备配置

仓库提供脱敏配置示例：

```text
data/config.example.json
```

复制为真实本地配置：

```bat
copy data\config.example.json data\config.json
```

macOS/Linux：

```bash
cp data/config.example.json data/config.json
```

打开 `data/config.json`，修改：

- `root_folder`：课程资料根目录。
- `server.host` / `server.port`：本地服务监听地址，默认 `127.0.0.1:8000`。
- `ai.base_url` / `ai.api_key` / `ai.model`：OpenAI-compatible LLM 配置。
- `ai.embedding_model` / `ai.embedding_dimensions`：可选 OpenAI-compatible embedding 配置，留空时使用本地 fallback。
- `mineru.token`：MinerU token。

真实配置文件位于 `data/config.json`，已经被 `.gitignore` 忽略，不应提交。

## 3. 准备课程资料

资料根目录下的一级文件夹就是课程名，例如：

```text
D:\StudyMaterials
├─ 操作系统
│  ├─ 教材
│  │  └─ chapter1.pdf
│  ├─ 课件
│  │  └─ process.pdf
│  └─ notes.md
├─ 数据结构
│  ├─ stack.pdf
│  └─ queue.txt
└─ 高等数学
   └─ limit.pdf
```

注意：

- PDF 建议小于 20 页，当前系统仍处于开发测试阶段。
- 课程资料可以包含子文件夹。
- 支持 PDF、TXT、MD、Markdown、DOCX 文件名识别，以及 PNG/JPG/WebP/GIF/BMP 图片预览；PDF、图片、文本和 Markdown 会在右侧预览区打开。DOCX 支持基础文本抽取用于入库，复杂版式、批注、页眉页脚、扫描件和图片文字仍建议转换为 PDF，或配合 MinerU 等更高质量解析工具。TXT 和 Markdown 预览可在当前文件内搜索并高亮定位。

## 4. 启动系统

Windows：

```bat
start.bat
```

macOS/Linux：

```bash
./start.sh
```

首次启动发现依赖缺失时，启动脚本会自动调用依赖安装器。后续启动只构建前端并运行服务。

浏览器访问：

```text
http://127.0.0.1:8000
```

如需改端口，可修改 `data/config.json`：

```json
{
  "server": {
    "host": "127.0.0.1",
    "port": 8010
  }
}
```

也可以临时设置环境变量 `COURSE_AGENT_HOST` / `COURSE_AGENT_PORT` 后再运行启动脚本。

截图参考：`docs/screenshots/01-首页总览.png`

## 5. 设置资料根目录

在顶部“资料根目录”输入框填写目录，例如：

```text
D:\StudyMaterials
```

点击“设置目录”或“刷新课程”。左侧会显示课程列表。

## 6. 浏览课程与文件

1. 点击左侧课程名。
2. 展开课程内部目录结构。
3. 点击 Markdown 文件查看渲染后的排版，点击 TXT 文件查看原始内容。
4. 在 TXT 或 Markdown 预览顶部输入关键词，可查看命中数，并用“上一个”“下一个”在当前文件内定位高亮结果。
5. 点击图片文件查看图片预览；点击 PDF 文件在右侧查看 PDF，并可通过链接在新窗口打开。

截图参考：

- `docs/screenshots/02-课程展开与文件树.png`
- `docs/screenshots/03-文本资料预览.png`

## 7. 构建课程知识库

选择课程后，点击“构建知识库”。系统会：

1. 清空当前课程旧索引。
2. 解析课程资料。
3. 将文本切片。
4. 写入 `data/indexes/<course_id>.json`。

每门课程单独构建知识库，避免不同课程资料混淆。

## 8. 使用课程 Agent

在右侧课程 Agent 中输入问题，点击“发送”。

支持模式：

- 答疑模式：直接回答。
- 启发模式：先给学习提示。
- 作业提示：只给思路和资料依据。
- 复习模式：适合整理重点和自测题。

回答会包含：

- 基于当前课程资料的答案。
- 来源文件、页码或片段编号。
- Agent trace：感知、读取、检索、回答、记忆。

截图参考：`docs/screenshots/04-课程Agent问答记录.png`

## 9. 拖入附件或截图提问

拖文件到聊天框后，系统会保存到：

```text
data/chat_uploads/<course_id>/
```

支持：

- PDF、Markdown、TXT 等文本资料。
- PNG、JPG、WebP、GIF、BMP 截图。

文本附件会被解析并加入当前课程检索；截图会作为图片内容块发送给已配置的 OpenAI-compatible 视觉模型。

## 10. 生成摘要和练习题

点击：

- “生成摘要”
- “生成练习题”

系统会生成 Markdown 文件并保存到当前课程目录：

```text
课程目录/AI生成/
```

生成后文件会出现在左侧课程文件树中，可以点击预览。`AI生成/` 目录在构建知识库时会被排除，避免模型生成内容再次成为后续回答的课程证据。

## 11. 管理课程笔记

点击右侧“笔记”按钮，打开课程笔记抽屉。

在抽屉中可以新增笔记，也可以对已保存笔记执行“编辑”“保存修改”“取消”或“删除”。

笔记保存到：

```text
data/course_memory/<course_id>/notes.json
```

后端也支持通过接口维护笔记：

- `POST /api/courses/{course_id}/notes` 新增笔记。
- `POST /api/courses/{course_id}/notes/{note_id}` 用 JSON `{title?, content?}` 修改笔记。
- `POST /api/courses/{course_id}/notes/{note_id}/delete` 删除笔记。

截图参考：`docs/screenshots/05-课程笔记抽屉.png`

## 12. 管理学习计划

选择课程后，左侧“学习计划”会显示该课程的完整计划、完成进度、下一项和剩余预计时间。

可以执行：

- 点击学习项左侧状态按钮，在“待学 / 进行中 / 完成”之间切换。
- 点击“编辑”，修改标题、类型和预计分钟数，再点击“保存”。
- 点击“删除”，移除不需要的学习项。
- 在“新增学习项”输入框中填写标题并点击加号，追加新的学习项。

学习计划保存到：

```text
data/course_memory/<course_id>/study_plan.json
```

后端接口：

- `GET /api/courses/{course_id}/plan` 获取学习计划。
- `POST /api/courses/{course_id}/plan` 新增学习项。
- `POST /api/courses/{course_id}/plan/{item_id}` 用 JSON `{title?, kind?, status?, estimated_minutes?}` 修改学习项。
- `POST /api/courses/{course_id}/plan/{item_id}/delete` 删除学习项。

## 13. 管理掌握度和错题

选择课程后，左侧“课程概览”中的“掌握度”区域会显示待复习数量、未订正错题数量、薄弱知识点和待复习知识点。

可以执行：

- 在“新增知识点”输入框填写知识点名称，必要时填写逗号分隔的别名，然后点击加号手动加入掌握度跟踪。
- 对薄弱或待复习知识点点击“对 / 错”，记录一次答题结果。
- 在未订正错题列表中查看题目、作答和参考答案，订正后点击“订正”标记为已订正。

掌握度状态保存到：

```text
data/course_memory/<course_id>/mastery.json
```

后端接口：

- `GET /api/courses/{course_id}/mastery` 获取完整掌握度状态。
- `POST /api/courses/{course_id}/mastery` 新增知识点或记录答题结果。
- `POST /api/courses/{course_id}/mastery/mistakes/{mistake_id}/resolve` 标记错题已订正。

## 14. 清空课程会话和记忆

每门课程的聊天和记忆都是文件：

```text
data/course_memory/<course_id>/
├─ messages.json
├─ memory.md
└─ notes.json
```

点击课程 Agent 工具栏里的“清空记忆”，确认后系统会清空当前课程的对话消息和课程记忆。也可以调用：

```text
POST /api/courses/{course_id}/memory/clear
```

接口会清空该课程的 `messages.json` 和 `memory.md`，并返回 `ok`、`messages`、`memory`。课程笔记不会被删除。
