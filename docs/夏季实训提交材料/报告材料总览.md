# 报告材料总览

## 1. 九章报告

- 完整正文：`docs/夏季实训提交材料/大型实习报告.md`
- 写作规范：`docs/报告撰写指南.md`

报告题目：

```text
基于 RAG 的 Windows 本地课程资料库与 AI 学习助手系统设计与实现
```

## 2. 需求和设计图

`系统设计.md` 和大型报告包含：总体架构、资料入库、多对话 ReAct、E-R 图、API、存储和安全边界。报告应突出当前能力：课程文件树、多对话、三种学习模式、hybrid RAG、引用、流式反馈、摘要、练习题和课程笔记。

学习计划不是当前公开能力；Dashboard/mastery 只有后端 API，当前前端不展示操作区。

## 3. 程序模块

使用 `程序模块与代码摘录.md` 中的当前代码：

- `CourseScanner.scan()` 课程识别。
- `ChatFlow.run()` 的 ReAct action。
- conversation 级消息/记忆边界。
- 旧单会话数据迁移。
- 引用支撑后处理。

不要引用旧 `server.chat()`、全局 `CTX` 或单一课程级 `messages.json`。
